38  /// <summary>
39  /// The smoothed orientation in space of the device as a Tetrad (Quaternion)
40  /// </summary>
41  /// <remarks>
42  /// Jose Rodríguez-Rosa and Jorge Martín-Gutiérrez / Procedia Computer Science 25 (2013) 436 – 442
43  /// <para/>
44  /// page 438:
45  /// <para/>
46  /// However, we encountered some issues when using gyroscope input data since it is very device dependent:
47  /// Sometimes gyroscope attitude readings are noisy, update slowly or require a quaternion correction if
48  /// the device screen orientation is changed (e.g. from Portrait to Landscape)[1].
49  /// <para/>
50  /// This was addressed using a quaternion spherical linear interpolation (slerp) between the current
51  /// camera attitude and the desired one (which might be multiplied by a second quaternion to correct the
52  /// screen orientation if needed) using a step factor of 0.2. This value was determined empirically and
53  /// effectively makes the interpolation to act as a low-pass filter, stabilizing the gyroscope reading
54  /// while keeping enough responsiveness for a good user experience. The sampling rate used for this sensor
55  /// was 15Hz (higher rates shortens device battery).
56  /// <para/>
57  /// In our tests, we found gyroscope on Apple iPhone smart-phone to be very stable and accurate over time;
58  /// on the other hand, Android it ́s very device dependent, but having the low-pass filter implemented as
59  /// stated above, effectively fixes this problem on most devices.
60  ///</remarks>
61  /// <a href="">The smoothed orientation in space of the device as a Tetrad (Quaternion)</a>
62  public Quaternion Attitude {
63  get {
64  rotateTo = Device.Attitude.RightToLeftHanded(Trig.ZAxis);
65  rotation = Quaternion.Slerp(rotateFrom, rotateTo, smoothing);
66  rotateFrom = rotateTo;
67  return rotation;
68  }
69  }
